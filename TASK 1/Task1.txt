1. Dbt merupakan sebuah tools open-source yang digunakan dalam menganalisis data dan dapat juga digunakan untuk transformasi data dalam proses merancang, menguji, dan mengeksekusi transformasi data dalam data warehouse. 

2. Lebih mudah dioperasikan, mdoel datanya fleksibel, sebagian proses transformasi ada di level Data Warehouse, ada proses built in testing data quality, learning curve rendah, dapat dijalankan dilingkungan cloud atau trigger airflow. Dengan kata lain DBT mampu memberikan kebebasandalam mengelola dependensi dengan dukungan tes secara realtime dan learning curve yang rendah sehingga memudahkan engineer dan non-engineer berkontribusi pada tahap transformasi. 

3. Dependency tree merupakan representasi grafis atau struktur model data dan hubungan antar data yang membantu memastikan setiap model data dijalankan dengan urutan yang benar dan hasilnya konsisten. Setiap model data dalam proyek DBT dapat bergantung pada model lain yang dicerminkan melalui dependency tree dengan menunjukan model mana yang harus dieksekusi terlebih dahulu. Sedangkan versioning merupakan pengelolaan versi dari model data dan kode transformasi yang memungkinkan pengguna untuk menjaga konsistensi dan melacak perkembangan model data seiring waktu. Dengan fungsi pengelolaan versi yang disediakan DBT pengguna dapat melacak setiap perubahan versi sebelumnya dan berkolaborasi dengan tim serta membuat snapshot versi saat ini yang memungkinkan engineer untuk mengelola eksperimen dan iterasi tanpa mengganggu stabilitas dan konsistensi.
